# Restaurant Menu & Order Management — Technical Implementation Plan

---

## Tech Stack Overview

| Layer | Technology | Purpose |
|---|---|---|
| Frontend | React 18 + Vite | Fast SPA build tool, HMR, modern React |
| Styling | Tailwind CSS v3 | Utility-first CSS, consistent spacing |
| UI Components | shadcn/ui | Accessible, customizable component primitives |
| Routing | React Router v6 | Client-side routing with protected routes |
| State Management | Zustand | Lightweight global state (cart, orders) |
| Backend / DB | Supabase | PostgreSQL database, Auth, Realtime, Storage |
| Server | Node.js + Express | Custom API for business logic, QR generation |
| QR Codes | `qrcode` npm package | Generate QR codes per table |
| Charts | Recharts | Sales dashboard visualisations |
| Tables | TanStack Table v8 | Sortable, filterable data tables |
| Drag & Drop | dnd-kit | Menu reordering |
| PDF / Export | `jsPDF` | Bill receipts, CSV export |
| Notifications | Sonner (shadcn) | Toast notifications |
| Realtime | Supabase Realtime | Live order updates to KDS and customer |
| Deployment | Vercel (frontend) + Railway (backend) | Or any Node-compatible host |

### Subphase 1.7 — Supabase Project Setup

**What you do manually in the Supabase Dashboard (supabase.com):**

1. Create a new Supabase project — give it name `restaurant-app`
2. Choose a region closest to your users
3. Set a strong database password — save it somewhere safe
4. Wait for project to provision (~2 minutes)
5. Go to **Settings → API**
6. Copy:
   - `Project URL` → goes into `client/.env.local` and `server/.env`
   - `anon public key` → goes into `client/.env.local` only
   - `service_role secret key` → goes into `server/.env` only (never expose this on frontend)
7. Go to **Authentication → Providers** — ensure Email is enabled
8. Go to **Authentication → Email Templates** — disable email confirmation for now (re-enable in production)

**Files to create manually in VSCode:**

`client/.env.local`:
```
VITE_SUPABASE_URL=your_project_url_here
VITE_SUPABASE_ANON_KEY=your_anon_key_here
VITE_API_URL=http://localhost:4000
```

`server/.env`:
```
SUPABASE_URL=your_project_url_here
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key_here
PORT=4000
JWT_SECRET=a_long_random_string_you_make_up
```

### Subphase 1.8 — Supabase Database Schema

**What you do manually:**
1. In Supabase Dashboard, go to **SQL Editor**
2. You will paste and run SQL scripts in this step (I will generate these when you reach this code step)

**Tables to be created (structure overview):**

- `restaurants` — restaurant name, logo, settings
- `tables` — table number, QR code URL, status, seats
- `menu_categories` — category name, display order, restaurant_id
- `menu_items` — name, description, price, image_url, category_id, dietary tags, available, is_special
- `item_variants` — variant name, price_modifier, item_id
- `item_addons` — addon name, price, item_id
- `orders` — table_id, status, created_at, total, customer_session_id
- `order_items` — order_id, item_id, quantity, notes, variant, addons, item_price
- `bill_requests` — order_id, split_count, tip_amount, email, created_at
- `feedback` — order_id, overall_rating, comment, created_at
- `profiles` — user_id (links to auth.users), role (owner/manager/kitchen/waiter), name
- `table_assignments` — table_id, profile_id (waiter assigned to table)

**Row Level Security (RLS) policies — what to manually enable:**
- Enable RLS on every table in Supabase Dashboard → Table Editor → each table → RLS toggle
- You will paste the RLS policy SQL in the code step for this phase

### Subphase 2.5 — Main Router Setup

**Files to create in VSCode:**
- `client/src/routes/AppRouter.jsx`

**Route structure to define:**
```
/menu/:tableId                  → CustomerMenuPage       (public, no auth)
/order-status/:sessionId        → OrderStatusPage        (public, session only)

/kitchen                        → KitchenDisplayPage     (auth: kitchen, manager, owner)
/manager/tables                 → TableManagementPage    (auth: manager, owner, waiter)
/manager/menu                   → MenuManagementPage     (auth: manager, owner)
/manager/reports                → ReportsDashboardPage   (auth: owner)
/manager/staff                  → StaffManagementPage    (auth: owner)
/not-authorized                 → NotAuthorizedPage      (public)
```

---

### Subphase 2.6 — Backend Auth Middleware

**Files to create in VSCode:**
- `server/src/middleware/auth.js`
- `server/src/middleware/roleCheck.js`

**What it does:**
- Extracts JWT from `Authorization: Bearer <token>` header
- Verifies it with Supabase or JWT_SECRET
- Attaches user object to `req.user`
- `roleCheck('owner', 'manager')` — restricts route to those roles

## Phase 4 — QR Code Generation

### Subphase 4.1 — QR Code Generation Endpoint

**Files to create in VSCode:**
- `server/src/routes/qr.js`

**What it does:**
- Manager triggers QR generation per table from the dashboard
- Backend uses `qrcode` npm package to generate a PNG dataURL
- QR encodes the URL: `https://yourdomain.com/menu/:tableId`
- QR image is uploaded to Supabase Storage bucket `qr-codes`
- Public URL is saved back to the `tables` record in Supabase

**Supabase Storage bucket to create manually:**
1. Go to Supabase Dashboard → Storage
2. Create bucket named `qr-codes` — set to Public
3. Create bucket named `menu-images` — set to Public

---

### Subphase 4.2 — QR Display & Download in Manager UI

## Important Notes

- Always start the backend before the frontend when developing locally: `cd server && npm run dev`, then `cd client && npm run dev`
- Never put `SUPABASE_SERVICE_ROLE_KEY` in the frontend `.env.local` — it bypasses all RLS
- Every time you add a new Supabase table, immediately enable RLS on it
- When you ask for code for each phase, specify the exact subphase number (e.g., "generate code for Phase 3 Subphase 3.4") for a clean step-by-step implementation
